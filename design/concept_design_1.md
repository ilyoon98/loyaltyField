# 으리챗 난투전 (가제) — 컨셉 기획서

> 갓필드(Godfield) 스타일 다인 개인전. 최대 8명+ 순차 턴제, 타겟 지명 공격, 캐릭터별 전용 효과.
> **이 문서는 "무엇을, 왜" 만드는가**(게임 규칙·재미·캐릭터·밸런스)를 다룬다.
> 데이터 구조·온라인 아키텍처 등 "어떻게 구현하는가"는 별도 **개발 기획서(dev_design.md)** 참조.

> **문서 분리 이력**: v10 통합 기획서를 컨셉/개발 둘로 분리. 카드 목록·캐릭터 콘텐츠 등 기획자가 만지는 데이터는 본 문서가 원본.

---

## 1. 게임 개요

| 항목 | 내용 |
|---|---|
| 장르 | 다인 개인전(Free-for-all) 턴제 아이템 난투 |
| 인원 | 2 ~ 8명 (개인전, 팀 없음) |
| 플랫폼 | 웹 (브라우저), 링크 공유로 플레이 |
| 승리 조건 | 마지막 1인 생존 (last man standing) |
| 1판 목표 시간 | 5~10분 |

### 핵심 재미 (이걸 검증한다)
- "누구를 때릴까"라는 **타겟 선택** 자체가 핵심 의사결정.
- 강한 사람이 두들겨 맞고(견제), 약자끼리 눈치 보고(연합), 막판에 뒤통수치는(배신·어부지리) 난투의 정치학.
- 캐릭터 전용 효과가 "누가 누구를 노리는가"에 변수를 더한다.

## 2. 기본 자원

| 자원 | 설명 | 기본값 |
|---|---|---|
| HP | 0이 되면 탈락 | 시작 50, 최대 99 |
| 손패(Hand) | 가진 아이템 카드 | 최대 6장, 시작 5장 |
| 아이템 풀 | 모든 플레이어가 공유하는 중앙 더미에서 랜덤 드로우 | 무한 |

> **설계 메모**: 공유 아이템 풀에서 랜덤으로 뽑는다. 개인 덱을 안 짜도 되므로 덱빌딩 부담이 없고, "이번에 뭐가 나오나"의 운 요소가 난투의 변수로 작동한다.

## 3. 아이템 분류

아이템은 크게 4종류 + 보조. 캐릭터 전용 효과는 이 분류 위에 얹는다.

### 3-1. 공격 (ATK)
타겟 1명을 지명해 데미지. 타겟은 방어/반사/튕기기 기회를 가진다.

### 3-2. 방어 (DEF)
공격당했을 때 사용. 데미지를 막거나 줄인다. **여러 장 누적 가능**(13-2).

### 3-3. 반사 (REFLECT)
공격당했을 때 사용. **데미지를 공격자에게 그대로 되돌린다.** 난투의 핵심 변수.

> **설계 메모**: "내가 A를 쳤는데 A가 반사 → 내가 맞음"이 발생. **연쇄 허용** — 반사된 공격을 공격자가 다시 막거나 반사할 수 있다. 안전 상한 20회(13-4). 반사는 희귀 자원으로 둬야 교착이 안 생긴다.

### 3-4. 전체공격 (ALL)
타겟 지명 없이 **자신을 제외한 전원**에게 발동. 대상별로 개별 명중 판정.

> **설계 메모**: "선두 주자 깎기"와 "동시 다발 견제"를 만드는 양념. **전체공격에도 반사·튕기기 가능**. 단 되받아치기는 항상 단일 대상.

### 3-5. 튕기기 (Bounce)
공격당했을 때 사용. 받은 데미지를 **랜덤 대상(생존자, 본인 포함)**에게 전달. 연쇄 가능. 본인에게 돌아오면 즉시 맞기.

### 3-6. 보조/회복 (HEAL 등)
HP 회복(자신 포함 대상 1명 선택), 추가 드로우 등.

## 4. 턴 흐름 (핵심 규칙)

### 4-1. 턴 순서
- 입장 순서로 고정된 순번. 1→2→…→N→1 반복.
- 탈락한 플레이어는 순번에서 **스킵**.
- 생존자 1명이 되면 게임 종료.

### 4-2. 한 플레이어의 턴 (액티브 플레이어 = AP)
```
1. 행동 선택 (택1):
   a) 공격 → 타겟 1명 지명
   b) 전체공격 → 타겟 자동(전원)
   c) 보조/회복 → 자신 포함 대상 선택
   d) 패스 → 카드 1장 획득
2. 행동 1회로 턴 종료. (턴당 1행동)
3. 다음 순번으로.
```

> **설계 메모(대기시간 대응)**: 다인 순차 턴제의 약점은 "내 차례 기다림". 완화책 둘 — ①턴당 1행동으로 각 턴을 짧게 ②남의 턴에도 내가 타겟이면 즉시 응답하므로 계속 개입.

### 4-3. 피격 처리 (남의 턴에 내가 타겟이 됐을 때)
타겟(TP)은 즉시 응답 선택 (제한시간 30초):
```
a) 방어 (여러 장 누적) → 데미지 무효/감소
b) 반사 → 공격자에게 적용 (공격자도 다시 응답 가능 — 연쇄)
c) 튕기기 → 랜덤 대상에게 전달 (연쇄 가능)
d) 그냥 맞기
→ HP 0 이하면 탈락
```

### 4-4. 전체공격 처리
```
1. AP가 전체공격 발동 (데미지 D)
2. AP 제외 전원이 각자 방어/반사/튕기기/맞기 선택 (반사·튕기기 결과는 단일 대상)
3. 각자 HP 정산, 0 이하 탈락
```

> 피격 응답·방어 중첩·연쇄의 상세 규칙은 13장 참조.

## 5. 캐릭터 시스템

### 5-1. 설계 원칙
- **기본 스탯(HP·손패 등)은 전원 동일.** 캐릭터로 인한 스탯 차이 없음.
- 차별점은 **"특정 카드를 쓸 때 발동하는 추가 효과(패시브)" 하나**뿐.
- 즉 **1 캐릭터 = 1 트리거 카드 + 1 추가 효과**.
- 같은 캐릭터 **중복 선택 허용** (방 채우기 쉬움).

> **밸런스 핵심**: 캐릭터 강함 = `트리거 카드 출현 빈도 × 추가 효과 크기`.
> 흔한 카드에 붙은 보너스는 자주 발동돼 강하고, 희귀 카드에 붙으면 약하다.
> → "흔한 카드 = 작은 보너스 / 희귀 카드 = 큰 보너스"로 균형.

### 5-2. 캐릭터 선택 흐름
```
1. 방 입장
2. 캐릭터 선택 (목록에서 1명, 중복 허용)
   - 각 캐릭터: 이름 / 컨셉 / 트리거 카드 / 추가 효과 표시
3. 전원 선택 완료 시 방장이 시작 (별도 준비 토글 없음 — 선택=준비)
```

### 5-3. 추가 효과 타입 (개념)
| type | 의미 | 발동 시점 |
|---|---|---|
| bonusDamage | 공격 데미지 +값 | 공격 시 |
| bonusDefense | 방어 시 추가 경감 +값 | 방어 시 |
| reflectBonus | 반사 데미지 +값 | 반사 시 |
| drawOnUse | 해당 카드 사용 시 카드 추가 드로우 | 사용 시 |
| healOnUse | 해당 카드 사용 시 HP 추가 회복 | 사용 시 |
| pierce | 해당 공격은 방어 무시 | 공격 시 |

> 데이터 구조(JSON)·효과 적용 위치는 개발 기획서 참조.

## 6. 으리챗 캐릭터 콘텐츠

> 전원 기본 스탯 동일. 차별점은 트리거 카드 1종 + 추가 효과 1개.
> **상태 표시**: ✅ = 코드에 트리거 배치 완료 / ⬜ = 기획만 확정, 카드 추가 시 코드 적용 예정.
> **일러스트**: 각 캐릭터에 카드 이미지가 들어갈 자리가 코드에 있음(현재 이모지 placeholder). 그림 준비되면 채운다.

| 상태 | 캐릭터 | 컨셉(특징) | 트리거 카드 | 발동 | 효과 | 비고 |
|---|---|---|---|---|---|---|
| ✅ | **최초** | 강아지 '뭉치'의 주인 — 든든한 동료 | 단검(흔함) | 공격 | bonusDamage +4 | 자주 발동, 소폭 보너스 |
| ✅ | **기종** | 허리 부상 — 버티는 탱커 | 갑옷(보통) | 방어 | bonusDefense +10 | 잘 버팀 |
| ✅ | **현일** | 운동광 — 힘세고 공격적 | 검(보통) | 공격 | bonusDamage +8 | 중간 보너스 |
| ✅ | **종원** | 요리사 — 회복 특화 | 물약(보통) | 사용 | healOnUse +15 | 회복량 최고 |
| ✅ | **기훈** | 게임 섭렵 — 노련한 변수 | 도끼(희귀) | 공격 | bonusDamage +15 | 드물게 발동, 강력 |
| ⬜ | **규형** | 토목·콘크리트 석사, 역마살 | 방패(보통) | 반사 | reflectBonus +8 | 단단하게 되받음 |
| ⬜ | **지원** | 모든 게임 잘함 — 만능형 | 행운(드로우) | 사용 | drawOnUse +1 | 손패 회전 빠름 |
| ⬜ | **소민** | 설계→경비원, 지키는 사람 | 방패(보통) | 방어 | drawOnUse +1 | 방어하며 자원 확보 |
| ⬜ | **준형** | 인싸 '준형전자', 견적의 달인 | 활(보통) | 공격 | pierce(방어 무시) | 약점을 찌름 |
| ⬜ | **종문** | 으리챗 막내 — 막내 보정 | 물약(보통) | 사용 | healOnUse +10 | 회복 빠름 |

> **미배치 5명(⬜)**: 트리거 카드(방패·활·행운 등)가 아직 카드 풀에 없어 코드 미적용. 해당 카드를 추가하면 배치된다.

### 6-1. 밸런스 점검 메모
- **자주 발동/약한 보너스**: 최초, 지원 → 꾸준함.
- **가끔 발동/강한 보너스**: 기훈(도끼 +15) → 한 방 로망, 발동 적어 OP 아님.
- **방어/생존형**: 기종·규형·소민 → 오래 살아남아 어부지리.
- **회복형**: 종문·종원 → 장기전 강함, 직접 공격 보너스 없음.
- **주의**: 반사는 희귀 자원이라 reflectBonus는 규형 1명만 → 교착 방지.

> **조정 여지**: 종문·종원 회복 컨셉 겹침(막내 보정 차별화 여지), 규형 역마살은 효과 미반영. 캐릭터 JSON·이미지 슬롯은 개발 기획서.

## 7. 카드 (Type 정의 + 카드 목록)

> 카드 목록은 **이 문서가 원본**(기획자가 밸런스 보며 수정). 데이터 컬럼 스펙·처리 규칙은 개발 기획서.

### 7-1. Type 정의
| Type | 동작 | 사용 시점 |
|---|---|---|
| ATK | 지명 1명 공격 | 내 턴 |
| ALL | 자신 제외 전원 공격 (대상별 개별 명중) | 내 턴 |
| HEAL | 자신 포함 대상 1명 회복 | 내 턴 |
| DEF | 들어온 데미지 무효/경감 (누적 가능) | 피격 시 |
| REFLECT | 데미지를 공격자에게 반사 (연쇄) | 피격 시 |
| Bounce | 데미지를 랜덤 대상에게 떠넘김 (연쇄, 본인이면 즉시 맞기) | 피격 시 |

### 7-2. 캐릭터 보너스 (Plus) 개념
- **ProbabilityUP**: 명중률 +값
- **ValueUP**: 기본 수치 +값
- **HEAL**: 효과에 더해 HP 회복
- **Target**: 동시 대상 수 +값 (튕기기 등)

### 7-3. 현재 카드 목록
| # | 이름 | Type | 수치 | 명중 | 시점 | 보너스 캐릭터 | 보너스 |
|---|---|---|---|---|---|---|---|
| 1 | 뭉치 짖기 | ALL | 10 | 50% | 내 턴 | 최초 | 명중 +25% |
| 2 | 칼빵 | ATK | 5 | 100% | 내 턴 | — | — |
| 3 | 요리하기 | HEAL | 5 | 100% | 내 턴 | 종원 | 회복 +2 |
| 4 | 진통제 | DEF | 10 | — | 피격 시 | 기종 | 방어 +5 |
| 5 | 프로틴 | ATK | 5 | 100% | 내 턴 | 현일 | 회복 +2 |
| 6 | 무지개 반사 | REFLECT | — | — | 피격 시 | — | — |
| 7 | 유수암쇄권 | Bounce | — | — | 피격 시 | 기훈 | 대상 +2 |

> **밸런스 메모**: 프로틴(때리며 회복)은 강력 → 보너스 작게. 튕기기는 자체 데미지 0, 피격 반응으로만 의미.
> **미배치 캐릭터(⬜)**: 규형·지원·소민·준형·종문의 트리거 카드(방패·활·행운 등)는 아직 카드 풀에 없음. 위 7장 카드는 현재 코드에 있는 7종 전부. 새 카드 추가 시 미배치 캐릭터가 배치된다.

## 8. 피격 응답·연쇄 규칙

> 구현 방식(큐 재귀 삽입 등)은 개발 기획서 13장. 여기선 **플레이어가 알아야 할 규칙**만.

### 8-1. 피격 응답 — 한 종류만
한 번의 피격에 아래 중 **한 가지**를 골라 확정.

| 응답 | 카드 | 결과 |
|---|---|---|
| 방어 | 1장 이상 누적 | 합산 방어량만큼 감소, ≥데미지면 0 |
| 반사 | 1장 | 공격자에게 되돌림 (연쇄) |
| 튕기기 | 1장 | 랜덤 생존자에게 전달 (연쇄), 본인이면 즉시 맞기 |
| 맞기 | 0장 | 그대로 수용 (연쇄 종료) |

**방어와 반사/튕기기는 섞지 않는다.**

### 8-2. 방어 중첩
방어 카드를 여러 장 누적: `최종데미지 = max(0, 데미지 − 방어 합산)`. 쓴 카드 전부 소모.

### 8-3. 연쇄
반사/튕기기로 되받아친 데미지에 받은 쪽이 다시 응답 가능 → 연쇄.
- 전체공격도 반사/튕기기 가능, 단 되받아치기는 단일 대상.
- **종료**: 응답 카드 없음 / 맞기 선택 / 사망 / 안전 상한 20회.
- **카드 보충은 연쇄 완전 종료 후** 1회 — 연쇄 중 손패 고정이 자연 종료를 보장.

## 9. 밸런스 초기값 (테스트하며 조정)

| 변수 | 초기값 | 조정 방향 |
|---|---|---|
| 시작 HP | 50 | 길면 ↓ |
| 최대 HP | 99 | 회복 상한 |
| 손패 최대 | 6 | 운 ↑ 원하면 ↓ |
| 강무기 출현율 | 낮게 | 너무 빨리 끝나면 ↓ |
| 방어:반사 비율 | 공격:방어:반사 ≈ 5:3:2 | 반사 많으면 교착 |
| 전체공격 비율 | 10% 이하 | 많으면 지명공격 정치성 죽음 |

> **메모**: 반사가 흔하면 아무도 공격 안 하는 교착이 다인전 붕괴 패턴. 반사는 희귀하게.
